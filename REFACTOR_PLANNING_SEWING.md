# Рефакторинг Planning и Sewing — Отчёт

## 1. Обнаруженные проблемы

| Проблема | Решение |
|----------|---------|
| Planning использовал недельное планирование (weekly) | Заменён на дневной календарь (calendar API) |
| Planning требовал модалку для редактирования | Inline editing прямо в ячейках |
| date_from / date_to вручную | Убраны, используется только месяц |
| Sewing — сложная карточная структура по этажам | Заменена на плоскую таблицу + боковая панель |
| Sewing — много вложенных форм | Панель «Задание на пошив» при выборе строки |
| Линия не была в фильтрах | Добавлена как «Этаж / Линия» в Planning |

## 4. Новый дизайн Sewing

```
┌────────────────────────────────────────┬─────────────────────────────────┐
│ Заказ   │ Модель │ Этаж │ Раскроено │ Доступно │ Сшито │ Печать           │
├─────────┼────────┼──────┼───────────┼──────────┼───────┼──────────────────┤
│ #1 ...  │ Пиджак │ 2    │ 300       │ 300      │ 0     │                  │
│ #2 ...  │ Куртка │ 3    │ 200       │ 50       │ 150   │ [Печать]         │  ← выбранная строка
└─────────┴────────┴──────┴───────────┴──────────┴───────┴──────────────────┘

При выборе строки (available > 0):
┌─────────────────────────────────────────────────────┐
│ Задание на пошив                                    │
│ Заказ: #2 Куртка                                    │
│ Раскроено: 200   Доступно: 50                       │
│                                                    │
│ Факт пошива:                                       │
│ 2025-03-10  [___]                                  │
│ 2025-03-11  [___]                                  │
│                                                    │
│ [Сохранить]  [Завершить → ОТК]  [Закрыть]          │
└─────────────────────────────────────────────────────┘
```

- available_for_sewing = cut_fact_qty − already_sewn_qty
- При available ≤ 0 строка недоступна (opacity 60%)
- Панель только для строк со статусом IN_PROGRESS

## 6. Производственный поток

```
Заказ
  → Закуп (procurement)
  → Планирование (production_plan_day)
  → Раскрой (cutting_tasks)
  → Пошив (sewing_fact → sewing_batches, status=READY_FOR_QC)
  → ОТК (qc_batches)
  → Склад
  → Отгрузка
```

- Планирование до раскроя ✓
- План по дням ✓
- Пошив ≤ раскроенного ✓
- READY_FOR_QC после завершения пошива ✓
