## 1. Найденные проблемы

| Проблема | Было | Стало |
|----------|------|-------|
| Планирование зависело от факта раскроя | calc-capacity возвращал 400 при отсутствии раскроя | Планирование независимо; план ставится до раскроя |
| Жёсткая валидация мощности 1000–5000 | Блокировался ввод любой другой мощности | Убрана; overload показывается в UI |
| Перегруз не отображался | Не было overload в week_totals | Добавлены overload, free_capacity |
| Пошив без учёта раскроя | Можно было ввести факт больше раскроенного | Валидация: факт ≤ actual_cut_qty |
| Planning UI — порядок по заказу | Сначала выбор модели, затем даты | Месячная матрица: месяц → цех → этаж |
| Sewing UI — много лишних элементов | Период, номер заказа, дубли кнопок | Оставлены: этаж, поиск, статус, основные действия |

- **Верхняя панель:** Месяц | Цех | Этаж | Поиск

## 4. Новый UX Sewing

- **Панель:** Этаж | Поиск | Статус | ОТК
- **Строка партии:** Заказчик, артикул, **Раскроено / Доступно к пошиву**
- **Ограничение:** факт не может превышать раскроенное; backend возвращает 400 при нарушении
- **Кнопки:** «Сохранить факт» | «Завершить → ОТК» | «Заказ»

## 5. Логика мощностей

```
capacity_week = capacity_override || weekly_capacity из БД
load_week = sum(planned по секции и неделе)
overload = max(0, load_week - capacity_week)
free_capacity = max(0, capacity_week - load_week)
```

- UI показывает перегруз красным
- Backend возвращает `overload`, `free_capacity` в `week_totals`
- Данные не обрезаются и не теряются

---

## 6. Flow: Planning → Cutting → Sewing → QC

1. **Заказ** создаётся
2. **Planning** — план по неделям (не зависит от раскроя)
3. **Cutting** — факт раскроя по этажу
4. **Sewing** — доступно только `actual_cut_qty - already_in_sewing`
5. **Завершить пошив** → SewingBatch со статусом **READY_FOR_QC**
6. **ОТК** — `batchesPendingQc` фильтрует по `READY_FOR_QC`
7. После ОТК партия идёт на склад
